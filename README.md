# Library-Information-System
This project aims to implement a database-backed Library Information System that allows users to search for books by title, genre, author, and stock. 
The system will store data on books, authors, users, and book
loaning. Librarians will be able to add new books to the system. The system will use primary keys and
foreign keys to ensure data integrity. We will write a simple interface that allows users to search for books
conveniently
Book Database and Library System:
This project aims to implement a database-backed Library Information System that allows users to search
for books by title, genre, author, and stock. The system will store data on books, authors, users, and book
loaning. Librarians will be able to add new books to the system. The system will use primary keys and
foreign keys to ensure data integrity. We will write a simple interface that allows users to search for books
conveniently.
1. End Users
The application is designed for:
● Students/Public: Users who want to search for books to read
● Library Staffs: Librarians who want to manage books
● System Maintainers/ IT staff: Maintain the database, manage user permissions
● University Researchers: Search for academic books and references
2. Capabilities of the Application
The system will include the following features:
● To provide book availability
● Search books by author name or title
● To provide book locations
● To provide easy access to books in different genres
● To organize books for easy access to readers and librarians
3. Software Used
● Database: SQL, PostgreSQL, or MySQL
● Front‑end: Python Flask or Node.js
● Development environment: VS Code, Git, SQLDeveloper
● Optional tools: Linux (Zeus)
4. Hardware Requirements
● Runs on standard CEC lab machines or Zeus.
● No special hardware required; only a web browser for the front‑end.
● Database hosted on Oracle server (CEC server)
